확장 기능 적용 가이드

라이브클래스 확장 기능은 관리자 페이지의 메타 영역에 코드를 붙여넣어 적용합니다.
아래 단계를 따라 진행하면 대부분의 기능을 직접 적용할 수 있습니다.

목차

  1. 사전 확인 — 코드 입력 기능 활성화
  2. 관리자 페이지 접속
  3. 메타 영역으로 이동
  4. 코드 붙여넣기
  5. 저장 & 확인
  6. 주의사항
STEP 0

사전 확인 — 코드 입력 기능 활성화

확장 기능을 적용하려면 관리자 페이지에서 코드 입력 기능이 활성화되어 있어야 합니다. 아래 경로에서 메타 영역을 찾을 수 없다면, 라이브클래스 고객성공팀에 활성화를 요청하세요.

코드 입력 기능이 보이지 않나요?
일부 사이트는 코드 입력이 기본 비활성화 상태입니다. 라이브클래스 채팅 상담 또는 고객성공팀에 "코드 입력 기능 활성화"를 요청하시면 됩니다.

STEP 1

관리자 페이지 접속

내 사이트의 관리자 페이지에 접속합니다.

내사이트.liveklass.com / admin

예시: example.liveklass.com/admin — 자신의 사이트 주소 뒤에 /admin을 붙이면 됩니다.

[스크린샷: 관리자 로그인 화면]
STEP 2

메타 영역으로 이동

관리자 페이지 왼쪽 메뉴에서 아래 경로를 따라 이동합니다.

검색 및 코드 설정 → 메타 영역
[스크린샷: 관리자 좌측 메뉴 > 검색 및 코드 설정]
[스크린샷: 메타 영역 입력 화면]

메타 영역이란?
사이트의 HTML 코드에 직접 삽입되는 영역입니다. 여기에 입력한 CSS/JS 코드는 사이트의 모든 페이지에 적용됩니다.

STEP 3

코드 붙여넣기

확장 기능 마켓에서 복사한 코드를 메타 영역에 붙여넣습니다. 코드 종류에 따라 감싸는 태그가 다릅니다.

CSS 코드인 경우

디자인 변경, 요소 숨김 등 CSS 코드는 <style> 태그로 감싸서 넣습니다.

<style> /* 예시: 신청 버튼 색상 변경 */ .btn-primary { background-color: #YOUR_COLOR !important; } </style>

JS 코드인 경우

텍스트 변경, 플로팅 CTA 등 JavaScript 코드는 <script> 태그로 감싸서 넣습니다.

<script> // 예시: 특정 텍스트 변경 const observer = new MutationObserver(() => { const el = document.querySelector('TARGET_SELECTOR'); if (el) el.textContent = '변경할 텍스트'; }); observer.observe(document.body, { childList: true, subtree: true }); </script>

코드에서 바꿔야 할 부분
#YOUR_COLOR → 원하는 색상 코드 (예: #ff6600)
TARGET_SELECTOR → 변경할 요소의 CSS 선택자
어떤 값을 넣어야 할지 모르겠으면 지원 요청을 보내주세요.

이미 다른 코드가 있다면?

메타 영역에 기존 코드(GA, GTM 등)가 있을 수 있습니다. 기존 코드를 지우지 말고, 아래쪽에 새 코드를 추가하세요.

STEP 4

저장 & 확인

코드를 붙여넣은 후 저장 버튼을 클릭하고, 내 사이트에서 결과를 확인합니다.

  • 메타 영역에 코드를 붙여넣고 저장 버튼 클릭
  • 내 사이트를 새 탭에서 열기 (또는 새로고침)
  • 변경사항이 적용되었는지 확인
  • 모바일에서도 확인 (디자인 변경의 경우)

변경이 안 보이나요?
브라우저 캐시 때문에 바로 반영되지 않을 수 있습니다. Ctrl+Shift+R (Mac: Cmd+Shift+R)로 강력 새로고침하세요.

주의사항

알아두면 좋은 것들

라이브클래스 업데이트 시

라이브클래스 플랫폼이 업데이트되면 CSS 선택자가 변경될 수 있습니다. 이 경우 적용했던 코드가 작동하지 않을 수 있으며, 선택자를 수정해야 합니다. 문제가 생기면 지원 요청을 보내주세요.

SPA 환경

라이브클래스는 SPA(Single Page Application) 방식으로 동작합니다. 페이지를 이동해도 전체 페이지가 새로 로드되지 않기 때문에, JavaScript 코드는 MutationObserver 패턴을 사용해야 합니다. 확장 기능 마켓의 코드 스니펫은 이미 이 패턴을 적용하고 있으니 안심하세요.

기존 코드와의 충돌

메타 영역에 이미 다른 코드가 있다면, 새 코드를 추가할 때 기존 코드를 삭제하지 마세요. 기존 코드 아래에 새 코드를 추가하면 됩니다. CSS 속성이 충돌할 경우 !important가 붙은 쪽이 우선 적용됩니다.

직접 적용이 어렵거나 막히는 부분이 있다면 언제든 요청해주세요.
